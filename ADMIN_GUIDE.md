# 🛠️ UTOPIA X 관리자 페이지 가이드

## 📋 개요

**UTOPIA X 관리자 페이지**는 아티스트 등록 신청, 섭외 요청, 댄서 DB를 관리할 수 있는 통합 대시보드입니다.

- **파일**: `admin.html`
- **접속 방법**: 브라우저에서 `admin.html` 열기
- **비밀번호**: `admin123` (개발용)
- **디자인**: 다크 모드 + 네온 그라데이션 (메인 사이트와 통일)

---

## 🔐 로그인

### 접근 방법
```
1. 브라우저에서 admin.html 열기
2. 비밀번호 입력: admin123
3. "로그인" 버튼 클릭
```

### 인증 시스템
- **방식**: localStorage 기반 간단 인증
- **세션 유지**: 브라우저를 닫아도 유지됨
- **로그아웃**: 좌측 하단 "로그아웃" 버튼

⚠️ **보안 주의**: 현재는 개발용입니다. 프로덕션에서는 실제 인증 시스템 필요!

## 📊 주요 기능 (우선순위별)

### 1. **대시보드 (E - 최우선)** ✅

#### 통계 카드 (4개)
- **총 댄서**: 현재 등록된 전체 댄서 수
- **섭외 요청**: 클라이언트 섭외 요청 총 개수
- **대기중 신청**: 승인 대기중인 아티스트 수
- **매칭 성공률**: 98% (현재 고정값)

#### 최근 활동
- 최근 5개 활동 표시
- 섭외 요청, 아티스트 등록 등
- 실시간 타임스탬프

#### 인기 태그
- AI 매칭에서 가장 많이 사용된 태그
- 태그별 사용 횟수 표시

---

### 2. **섭외 요청 관리 (C - 우선순위 2)** ✅

#### 기능
- 모든 섭외 요청 카드 형태로 표시
- 클라이언트 정보 (이름, 이메일, 연락처)
- 행사 정보 (날짜, 유형, 필요 댄서 수, 예산)
- AI 프롬프트 미리보기

#### 상세 보기
- 카드 클릭 시 모달로 전체 정보 표시
- AI 분석 결과 (hardFilters + softScores)
- 추천된 댄서 목록 (향후 구현)

#### 검색 & 필터
- 검색: 클라이언트 이름, 이메일 등
- 필터: 전체/신규/검토중/완료

---

### 3. **아티스트 등록 신청 관리 (A - 우선순위 3)** ✅

#### 기능
- 모든 등록 신청 카드 형태로 표시
- 상태 배지: 대기중(노란색) / 승인(초록색) / 거부(빨간색)
- 기본 정보: 이름, 이메일, 연락처, 전문 장르
- 비주얼 정보: 성별, 키, 체형 등

#### 승인 프로세스
```
1. 카드에서 "승인" 버튼 클릭
   또는 카드 클릭 → 상세 보기 → 승인

2. 승인 시:
   - 상태가 "승인됨"으로 변경
   - 댄서 DB에 자동 추가 (향후 구현)
   - 이메일 알림 발송 (향후 구현)

3. 거부 시:
   - 상태가 "거부됨"으로 변경
   - 거부 사유 입력 (향후 구현)
```

#### 상세 보기
- 63개 전체 필드 표시
- 비주얼 프로필 (P0/P1/P2)
- 스킬 정보 (연기력, 가창력 등)
- 특수 태그 (어린이 친화, 특수분장 가능 등)
- 영상 링크

---

### 4. **빠른섭외 댄서 관리 (B - 우선순위 4)** 🚧

#### 기능 (향후 구현)
- 메인 페이지 "빠른섭외" 리스트 관리
- 드래그 앤 드롭으로 순서 변경
- 최대 10명 선택
- 전체 댄서 DB에서 추가/삭제

#### 구현 예정
```javascript
// SortableJS 라이브러리 활용
const sortable = new Sortable(document.getElementById('featuredDancersList'), {
    animation: 150,
    handle: '.drag-handle',
    onEnd: function(evt) {
        // 순서 변경 저장
    }
});
```

---

### 5. **댄서 DB 관리 (D)** ✅

#### 기능
- 전체 댄서 카드 형태로 표시
- 이미지, 이름, 전문 장르, 평점 등
- 검색 & 필터 (장르별)

#### 향후 추가 기능
- 댄서 추가/수정/삭제
- 이미지 업로드
- 63개 필드 전체 수정
- 태그 수동 조정

## 🎨 UI/UX 가이드

### 레이아웃 구조
```
┌─────────────────────────────────────────────┐
│ [Sidebar]           [Header]                │
│  로고               섹션 제목 | 시간 | 관리자│
│  대시보드           ─────────────────────────│
│  섭외 요청          [Main Content]          │
│  아티스트 신청      통계 카드 / 카드 그리드  │
│  빠른섭외 관리                              │
│  댄서 DB                                    │
│                                             │
│  [로그아웃]                                 │
└─────────────────────────────────────────────┘
```

### 색상 시스템
- **배경**: 검정 (#000000)
- **카드**: 어두운 회색 (#111111)
- **강조**: 핑크-퍼플-블루 그라데이션
- **상태 색**:
  - 대기중: 노란색 (#F59E0B)
  - 승인: 초록색 (#10B981)
  - 거부: 빨간색 (#EF4444)
  - 신규: 파란색 (#3B82F6)

### 카드 디자인
- **호버 효과**: 위로 5px 이동 + 보라색 테두리 + 그림자
- **상태 배지**: 좌측 상단 고정
- **버튼**: 그라데이션 (Primary) / 투명 (Secondary)

## 🔄 데이터 흐름

### 데이터 로딩 순서
```
1. 로그인 성공
   ↓
2. loadAllData() 호출
   ↓
3. 병렬 로딩:
   - loadDancersData()
   - loadCastingRequestsData()
   - loadArtistApplicationsData()
   ↓
4. 대시보드 통계 업데이트
   ↓
5. 최근 활동 렌더링
```

### API 엔드포인트
```javascript
// 댄서 DB
GET /tables/dancers?limit=100

// 섭외 요청
GET /tables/casting_requests?limit=100

// 아티스트 등록
GET /tables/artist_registrations?limit=100
```

## 🧪 테스트 방법

### 1. 로그인 테스트
```
1. admin.html 열기
2. 비밀번호: admin123 입력
3. 로그인 성공 → 대시보드 표시 확인
```

### 2. 데이터 로딩 테스트
```
1. F12 → Console 열기
2. 로그인 후 콘솔에서 확인:
   - "Loading dancers..."
   - "Loading casting requests..."
   - "Loading artist applications..."
3. 통계 카드 숫자 업데이트 확인
```

### 3. 섹션 전환 테스트
```
1. 좌측 메뉴 클릭
2. 각 섹션 정상 표시 확인
3. 헤더 제목 변경 확인
4. 활성 메뉴 하이라이트 확인
```

### 4. 카드 클릭 테스트
```
1. 섭외 요청 카드 클릭 → 모달 열림
2. 아티스트 신청 카드 클릭 → 상세 정보 표시
3. 모달 닫기 (X 버튼 / ESC / 외부 클릭)
```

1. **대시보드에서 전체 현황 파악**

4. **빠른섭외 리스트 주기 업데이트**
